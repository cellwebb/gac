# GAC के साथ Claude Code का उपयोग करना

[English](../en/CLAUDE_CODE.md) | [简体中文](../zh-CN/CLAUDE_CODE.md) | [繁體中文](../zh-TW/CLAUDE_CODE.md) | [日本語](../ja/CLAUDE_CODE.md) | [한국어](../ko/CLAUDE_CODE.md) | **हिन्दी** | [Tiếng Việt](../vi/CLAUDE_CODE.md) | [Français](../fr/CLAUDE_CODE.md) | [Русский](../ru/CLAUDE_CODE.md) | [Español](../es/CLAUDE_CODE.md) | [Português](../pt/CLAUDE_CODE.md) | [Norsk](../no/CLAUDE_CODE.md) | [Svenska](../sv/CLAUDE_CODE.md) | [Deutsch](../de/CLAUDE_CODE.md) | [Nederlands](../nl/CLAUDE_CODE.md) | [Italiano](../it/CLAUDE_CODE.md)

GAC Claude Code सदस्यता के माध्यम से प्रमाणीकरण का समर्थन करता है, जिससे आप महंगे Anthropic API का भुगतान करने के बजाय अपनी Claude Code सदस्यता का उपयोग कर सकते हैं। यह उन उपयोगकर्ताओं के लिए उत्तम है जिनके पास पहले से ही अपनी सदस्यता के माध्यम से Claude Code तक पहुंच है।

## Claude Code क्या है?

Claude Code Anthropic की सदस्यता सेवा है जो OAuth-आधारित Claude मॉडल पहुंच प्रदान करती है। API कुंजियों (जो टोकन-प्रति बिल की जाती हैं) का उपयोग करने के बजाय, Claude Code आपकी सदस्यता से OAuth टोकन का उपयोग करता है।

## लाभ

- **लागत प्रभावी**: अलग से API पहुंच के लिए भुगतान करने के बजाय अपनी मौजूदा Claude Code सदस्यता का उपयोग करें
- **समान मॉडल**: समान Claude मॉडल्स तक पहुंच (जैसे `claude-sonnet-4-5`)
- **अलग बिलिंग**: Claude Code का उपयोग Anthropic API बिलिंग से अलग है

## सेटअप

GAC में Claude Code के लिए अंतर्निहित OAuth प्रमाणीकरण शामिल है। सेटअप प्रक्रिया पूरी तरह से स्वचालित है और प्रमाणीकरण के लिए आपका ब्राउज़र खोलेगी।

### विकल्प 1: प्रारंभिक सेटअप के दौरान (अनुशंसित)

`gac init` चलाते समय, बस "Claude Code" को अपने प्रदाता के रूप में चुनें:

```bash
gac init
```

विज़ार्ड निम्न कार्य करेगा:

1. आपको प्रदाता सूची से "Claude Code" चुनने के लिए कहेगा
2. OAuth प्रमाणीकरण के लिए स्वचालित रूप से आपका ब्राउज़र खोलेगा
3. आपके एक्सेस टोकन को `~/.gac.env` में सहेजेगा
4. डिफ़ॉल्ट मॉडल सेट करेगा

### विकल्प 2: बाद में Claude Code पर स्विच करें

यदि आपके पास पहले से ही दूसरे प्रदाता के साथ GAC कॉन्फ़िगर है और आप Claude Code पर स्विच करना चाहते हैं:

```bash
gac model
```

फिर:

1. प्रदाता सूची से "Claude Code" चुनें
2. OAuth प्रमाणीकरण के लिए आपका ब्राउज़र स्वचालित रूप से खुलेगा
3. टोकन `~/.gac.env` में सहेजा गया
4. मॉडल स्वचालित रूप से कॉन्फ़िगर किया गया

### GAC को सामान्य रूप से उपयोग करें

एक बार प्रमाणीकृत होने के बाद, GAC को जैसे आम तौर पर उपयोग करें:

```bash
# अपने परिवर्तनों को स्टेज करें
git add .

# Claude Code के साथ जेनरेट और कमिट करें
gac

# या एकल कमिट के लिए मॉडल को ओवरराइड करें
gac -m claude-code:claude-sonnet-4-5
```

## उपलब्ध मॉडल

Claude Code Anthropic API के समान मॉडल तक पहुंच प्रदान करता है। वर्तमान Claude 4.5 परिवार मॉडल में शामिल हैं:

- `claude-sonnet-4-5` - नवीनतम और सबसे बुद्धिमान Sonnet मॉडल, कोडिंग के लिए सर्वोत्तम
- `claude-haiku-4-5` - तेज़ और कुशल
- `claude-opus-4-5` - जटिल तर्क के लिए सबसे सक्षम मॉडल

उपलब्ध मॉडल की पूरी सूची के लिए [Claude दस्तावेज़](https://docs.claude.com/en/docs/about-claude/models/overview) देखें।

## समस्या निवारण

### टोकन समाप्त हो गया

यदि आपको प्रमाणीकरण त्रुटियां दिखाई देती हैं, तो आपका टोकन समाप्त हो गया हो सकता है। निम्नलिखित चलाकर फिर से प्रमाणित करें:

```bash
gac auth claude-code login
```

आपका ब्राउज़र नए OAuth प्रमाणीकरण के लिए स्वचालित रूप से खुल जाएगा। वैकल्पिक रूप से, आप `gac model` चला सकते हैं, "Claude Code (OAuth)" का चयन कर सकते हैं, और "फिर से प्रमाणित करें (नया टोकन प्राप्त करें)" चुन सकते हैं।

### प्रमाणीकरण स्थिति जांचें

यह जांचने के लिए कि आप वर्तमान में प्रमाणीकृत हैं या नहीं:

```bash
gac auth claude-code status
```

या सभी प्रदाताओं को एक साथ जांचें:

```bash
gac auth
```

### लॉगआउट

अपना संग्रहीत टोकन हटाने के लिए:

```bash
gac auth claude-code logout
```

### "CLAUDE_CODE_ACCESS_TOKEN नहीं मिला"

इसका मतलब है कि GAC आपका एक्सेस टोकन नहीं ढूंढ सकता। निम्न चलाकर प्रमाणीकृत करें:

```bash
gac model
```

फिर प्रदाता सूची से "Claude Code" चुनें। OAuth प्रवाह स्वचालित रूप से शुरू होगा।

### "प्रमाणीकरण विफल"

यदि OAuth प्रमाणीकरण विफल होता है:

1. सुनिश्चित करें कि आपके पास एक सक्रिय Claude Code सदस्यता है
2. जांचें कि आपका ब्राउज़र सही ढंग से खुलता है
3. यदि समस्याएं बनी रहती हैं तो एक अलग ब्राउज़र आज़माएं
4. `claude.ai` के लिए नेटवर्क कनेक्टिविटी सत्यापित करें
5. स्थानीय कॉलबैक सर्वर के लिए पोर्ट 8765-8795 उपलब्ध हैं या नहीं जांचें

## Anthropic प्रदाता से अंतर

| सुविधा       | Anthropic (`anthropic:`)        | Claude Code (`claude-code:`)                                               |
| ------------ | ------------------------------- | -------------------------------------------------------------------------- |
| प्रमाणीकरण   | API कुंजी (`ANTHROPIC_API_KEY`) | OAuth (स्वचालित ब्राउज़र प्रवाह)                                           |
| बिलिंग       | प्रति-टोकन API बिलिंग           | सदस्यता-आधारित                                                             |
| सेटअप        | मैनुअल API कुंजी प्रविष्टि      | `gac init` या `gac model` के माध्यम से स्वचालित OAuth                      |
| टोकन प्रबंधन | दीर्घकालिक API कुंजी            | OAuth टोकन (समाप्त हो सकते हैं, `model` के माध्यम से आसान पुनः प्रमाणीकरण) |
| मॉडल         | समान मॉडल                       | समान मॉडल                                                                  |

## सुरक्षा नोट्स

- **कभी भी अपने एक्सेस टोकन को वर्जन कंट्रोल में कमिट न करें**
- GAC स्वचालित रूप से टोकन को `~/.gac.env` (आपकी प्रोजेक्ट निर्देशिका के बाहर) में संग्रहीत करता है
- टोकन समाप्त हो सकते हैं और `gac model` के माध्यम से पुनः प्रमाणीकरण की आवश्यकता हो सकती है
- OAuth प्रवाह बढ़ी हुई सुरक्षा के लिए PKCE (Proof Key for Code Exchange) का उपयोग करता है
- स्थानीय कॉलबैक सर्वर केवल localhost पर चलता है (पोर्ट 8765-8795)
