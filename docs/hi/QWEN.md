# GAC के साथ Qwen.ai का उपयोग करना

GAC, Qwen.ai OAuth के माध्यम से प्रमाणीकरण का समर्थन करता है, जिससे आप कमिट संदेश जनरेशन के लिए अपने Qwen.ai खाते का उपयोग कर सकते हैं। यह एक सहज लॉगिन अनुभव के लिए OAuth डिवाइस फ्लो प्रमाणीकरण का उपयोग करता है।

## Qwen.ai क्या है?

Qwen.ai अलीबाबा क्लाउड का AI प्लेटफ़ॉर्म है जो Qwen परिवार के बड़े भाषा मॉडल तक पहुँच प्रदान करता है। GAC OAuth-आधारित प्रमाणीकरण का समर्थन करता है, जिससे आप API कुंजियों को मैन्युअल रूप से प्रबंधित किए बिना अपने Qwen.ai खाते का उपयोग कर सकते हैं।

## लाभ

- **आसान प्रमाणीकरण**: OAuth डिवाइस फ्लो - बस अपने ब्राउज़र से लॉग इन करें
- **कोई API कुंजी प्रबंधन नहीं**: प्रमाणीकरण स्वचालित रूप से संभाला जाता है
- **Qwen मॉडल तक पहुँच**: कमिट संदेश जनरेशन के लिए शक्तिशाली Qwen मॉडल का उपयोग करें

## सेटअप

GAC में डिवाइस फ्लो का उपयोग करके Qwen.ai के लिए अंतर्निहित OAuth प्रमाणीकरण शामिल है। सेटअप प्रक्रिया एक कोड प्रदर्शित करेगी और प्रमाणीकरण के लिए आपका ब्राउज़र खोलेगी।

### विकल्प 1: प्रारंभिक सेटअप के दौरान (अनुशंसित)

`gac init` चलाते समय, बस प्रदाता सूची से "Qwen.ai (OAuth)" चुनें:

```bash
gac init
```

विज़ार्ड करेगा:

1. आपसे प्रदाता सूची से "Qwen.ai (OAuth)" चुनने के लिए कहेगा
2. एक डिवाइस कोड प्रदर्शित करेगा और आपका ब्राउज़र खोलेगा
3. आप Qwen.ai पर प्रमाणित करेंगे और कोड दर्ज करेंगे
4. अपने एक्सेस टोकन को सुरक्षित रूप से सहेजें
5. डिफ़ॉल्ट मॉडल सेट करें

### विकल्प 2: बाद में Qwen.ai पर स्विच करें

यदि आपने पहले से ही किसी अन्य प्रदाता के साथ GAC कॉन्फ़िगर किया है और Qwen.ai पर स्विच करना चाहते हैं:

```bash
gac model
```

फिर:

1. प्रदाता सूची से "Qwen.ai (OAuth)" चुनें
2. डिवाइस कोड प्रमाणीकरण प्रवाह का पालन करें
3. टोकन `~/.gac/oauth/qwen.json` में सुरक्षित रूप से सहेजा गया
4. मॉडल स्वचालित रूप से कॉन्फ़िगर किया गया

### विकल्प 3: सीधा लॉगिन

आप इसका उपयोग करके सीधे भी प्रमाणित कर सकते हैं:

```bash
gac auth qwen login
```

यह करेगा:

1. एक डिवाइस कोड प्रदर्शित करें
2. Qwen.ai प्रमाणीकरण पृष्ठ पर अपना ब्राउज़र खोलें
3. आपके प्रमाणित करने के बाद, टोकन स्वचालित रूप से सहेजा जाता है

### GAC का सामान्य रूप से उपयोग करें

एक बार प्रमाणित हो जाने पर, GAC का सामान्य रूप से उपयोग करें:

```bash
# अपने परिवर्तनों को स्टेज करें
git add .

# Qwen.ai के साथ जनरेट और कमिट करें
gac

# या एकल कमिट के लिए मॉडल को ओवरराइड करें
gac -m qwen:qwen3-coder-plus
```

## उपलब्ध मॉडल

Qwen.ai OAuth एकीकरण उपयोग करता है:

- `qwen3-coder-plus` - कोडिंग कार्यों के लिए अनुकूलित (डिफ़ॉल्ट)

यह portal.qwen.ai OAuth एंडपॉइंट के माध्यम से उपलब्ध मॉडल है। अन्य Qwen मॉडल के लिए, OpenRouter प्रदाता का उपयोग करने पर विचार करें जो अतिरिक्त Qwen मॉडल विकल्प प्रदान करता है।

## प्रमाणीकरण आदेश

GAC Qwen.ai प्रमाणीकरण के प्रबंधन के लिए कई आदेश प्रदान करता है:

```bash
# Qwen.ai में लॉगिन करें
gac auth qwen login

# प्रमाणीकरण स्थिति की जाँच करें
gac auth qwen status

# लॉगआउट करें और संग्रहीत टोकन हटा दें
gac auth qwen logout

# सभी OAuth प्रदाताओं की स्थिति की जाँच करें
gac auth
```

### लॉगिन विकल्प

```bash
# मानक लॉगिन (स्वचालित रूप से ब्राउज़र खोलता है)
gac auth qwen login

# ब्राउज़र खोले बिना लॉगिन करें (मैन्युअल रूप से जाने के लिए URL प्रदर्शित करता है)
gac auth qwen login --no-browser

# शांत मोड (न्यूनतम आउटपुट)
gac auth qwen login --quiet
```

## समस्या निवारण

### टोकन समाप्त हो गया

यदि आपको प्रमाणीकरण त्रुटियां दिखाई देती हैं, तो आपका टोकन समाप्त हो सकता है। इसे चलाकर पुनः प्रमाणित करें:

```bash
gac auth qwen login
```

डिवाइस कोड फ्लो शुरू हो जाएगा, और आपका ब्राउज़र पुनः प्रमाणीकरण के लिए खुल जाएगा।

### प्रमाणीकरण स्थिति की जाँच करें

यह जाँचने के लिए कि क्या आप वर्तमान में प्रमाणित हैं:

```bash
gac auth qwen status
```

या एक साथ सभी प्रदाताओं की जाँच करें:

```bash
gac auth
```

### लॉगआउट

अपने संग्रहीत टोकन को हटाने के लिए:

```bash
gac auth qwen logout
```

### "Qwen प्रमाणीकरण नहीं मिला"

इसका मतलब है कि GAC आपका एक्सेस टोकन नहीं ढूँढ सकता। इसे चलाकर प्रमाणित करें:

```bash
gac auth qwen login
```

या `gac model` चलाएं और प्रदाता सूची से "Qwen.ai (OAuth)" चुनें।

### "प्रमाणीकरण विफल"

यदि OAuth प्रमाणीकरण विफल हो जाता है:

1. सुनिश्चित करें कि आपके पास Qwen.ai खाता है
2. जाँचें कि आपका ब्राउज़र सही ढंग से खुलता है
3. सत्यापित करें कि आपने डिवाइस कोड सही ढंग से दर्ज किया है
4. यदि समस्या बनी रहती है तो कोई अलग ब्राउज़र आज़माएँ
5. `qwen.ai` के लिए नेटवर्क कनेक्टिविटी सत्यापित करें

### डिवाइस कोड काम नहीं कर रहा है

यदि डिवाइस कोड प्रमाणीकरण काम नहीं कर रहा है:

1. सुनिश्चित करें कि कोड समाप्त नहीं हुआ है (कोड सीमित समय के लिए मान्य हैं)
2. ताज़ा कोड के लिए फिर से `gac auth qwen login` चलाने का प्रयास करें
3. यदि ब्राउज़र खोलना विफल हो जाता है तो `--no-browser` फ़्लैग का उपयोग करें और मैन्युअल रूप से URL पर जाएँ

## सुरक्षा नोट्स

- **कभी भी अपने एक्सेस टोकन को** संस्करण नियंत्रण में कमिट न करें
- GAC स्वचालित रूप से टोकन को `~/.gac/oauth/qwen.json` (आपकी प्रोजेक्ट निर्देशिका के बाहर) में संग्रहीत करता है
- टोकन फ़ाइलों में प्रतिबंधित अनुमतियाँ हैं (केवल स्वामी द्वारा पठनीय)
- टोकन समाप्त हो सकते हैं और पुनः प्रमाणीकरण की आवश्यकता होगी
- OAuth डिवाइस फ्लो को हेडलेस सिस्टम पर सुरक्षित प्रमाणीकरण के लिए डिज़ाइन किया गया है
